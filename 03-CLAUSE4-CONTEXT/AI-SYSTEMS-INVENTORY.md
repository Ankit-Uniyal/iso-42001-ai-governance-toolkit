## Purpose

This inventory captures all AI systems used by or on behalf of the organisation. It is the foundation of the AIMS scope and feeds into risk assessment, impact assessment, and monitoring activities.

**AI System definition (per ISO 42001):** A machine-based system that, for a given set of objectives, makes predictions, recommendations, decisions, or content influencing real or virtual environments.

## Field Definitions

| Field | Guidance |
|-------|----------|
| System ID | Unique identifier (e.g., AI-001) |
| System Name | Descriptive name of the system |
| Business Function | Department or process the system supports |
| AI Type | e.g., ML classification, NLP, computer vision, generative AI, recommendation engine |
| Purpose / Use Case | What decision or action the system drives |
| Owner | Named individual or team accountable for the system |
| Risk Level | Critical / High / Medium / Low — based on impact if system fails or produces harmful output |
| In AIMS Scope | Whether this system is covered by the AIMS |
| Deployment Status | Live / Pilot / Development / Decommissioned |
| Last Assessment Date | Date of most recent AI System Impact Assessment (Clause 8.2) |

---

## Risk Level Criteria

| Level | Description |
|-------|-------------|
| Critical | High-stakes decisions affecting individual rights, safety, or significant financial harm |
| High | Significant operational impact; decisions affecting many users or sensitive data |
| Medium | Moderate impact; limited user exposure; human review available |
| Low | Low stakes, minimal personal data, easily reversible decisions |
