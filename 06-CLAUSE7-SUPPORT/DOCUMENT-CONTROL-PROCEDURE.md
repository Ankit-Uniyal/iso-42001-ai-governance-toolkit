## 1. Purpose

This procedure defines how all documented information within the AIMS is created, approved, versioned, distributed, and retired. It ensures that AIMS documents are current, accessible, and protected from unintended alteration or loss.

---

## 2. Scope

This procedure applies to all documented information required by ISO/IEC 42001:2023 and any additional documents determined necessary for effective AIMS operation, as listed in the Master Document List (AIMS-MDL-001).

---

## 3. Document Identification Standards

Every AIMS document must include:

| Field | Requirement |
|-------|-------------|
| Document ID | Unique identifier (e.g., AIMS-XXX-001) |
| Document Title | Descriptive, unambiguous title |
| Version | Version number (e.g., 1.0, 1.1, 2.0) |
| Owner | Named individual responsible for the document |
| Approved By | Name and role of approving authority |
| Date | Date of current version |
| Review Date | Scheduled next review date |
| Classification | e.g., Internal / Confidential / Public |
| Status | Active / Under Review / Superseded / Withdrawn |

---

## 4. Document Lifecycle

### 4.1 Creation
- Document owner drafts using approved template
- New documents assigned a unique Document ID by the AI Governance Lead
- Added to Master Document List before approval

### 4.2 Review and Approval
- Draft circulated to relevant stakeholders for review
- Comments collated and addressed
- Final version submitted to approving authority
- Approval recorded (signature or tracked email/system approval)

### 4.3 Publication and Distribution
- Published to central document repository (SharePoint / Confluence / GitHub)
- Communicated to relevant staff via the Communication Plan
- Previous version archived as superseded

### 4.4 Version Control
- Minor changes: increment minor version (e.g., 1.0 to 1.1)
- Major revisions: increment major version (e.g., 1.1 to 2.0)
- Changes documented in the document's revision history table
- All versions retained per the Records Retention Schedule

### 4.5 Periodic Review
- All documents reviewed on their scheduled review date (minimum annual)
- Owner confirms document remains accurate and fit for purpose
- Updated if necessary; version incremented if changes made
- Review date updated in Master Document List

### 4.6 Withdrawal / Retirement
- Document marked Withdrawn in Master Document List
- Withdrawn documents removed from active circulation
- Retained per retention schedule then securely destroyed

## 5. Document Repository

| Repository | Access | Document Types Stored |
|-----------|--------|----------------------|
| [SharePoint / Confluence / GitHub] | All in-scope staff (read); AI Gov Lead (write) | All AIMS documents |
| Audit evidence folder | Internal Auditor + AI Gov Lead | Audit reports, NCRs |
| HR system | HR + AI Gov Lead | Training records, competence evidence |

---

## 6. Protecting Documented Information

- Documents classified as Confidential accessible only to authorised staff
- No AIMS documents stored on personal devices without encryption
- Access rights reviewed annually
- Version history enabled on all document repositories
