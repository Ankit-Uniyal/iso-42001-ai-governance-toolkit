## Programme Objectives

This audit programme covers all clauses of ISO/IEC 42001:2023 and all in-scope AI systems at least once per cycle. It ensures that the AIMS conforms to the organisation's own requirements, to ISO 42001 requirements, and that it is effectively implemented and maintained.

## Audit Methods

| Method | When Used |
|--------|----------|
| Document review | All audits — review policies, procedures, records against requirements |
| Interviews | All audits — confirm understanding and implementation |
| Process observation | Operations audits — observe AI lifecycle, deployment, monitoring |
| Data sampling | Risk and performance audits — sample risk records, training records |
| System testing | AI system audits — verify technical controls |

---

## Audit Findings Classification

| Class | Description | Response Required |
|-------|-------------|-----------------|
| Major Nonconformity | Complete failure to meet a requirement; AIMS effectiveness compromised | Root cause analysis and corrective action within 30 days |
| Minor Nonconformity | Partial implementation; isolated failure | Corrective action within 60 days |
| Observation | Area for improvement; not yet a nonconformity | Consider improvement action |
| Conformance | Requirement is met and evidence is adequate | No action required |
